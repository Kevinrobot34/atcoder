## D - An Ordinary Game (500点)
* keywords
  - ゲーム系、後ろから考える
* 解法
  - 直接最適な行動を考えてシミュレーションするとかはしんどそう
  - 最終的な状態を考える
    - `s[0] == s[-1]` の時
      - `'ababa'` 的な長さ **奇数** の文字列になって終わる
      - もしこの状態が初期状態であれば答えは'Second'
      - そうで無い場合、そこまで至るのに必要な回数の偶奇が大事
        - `len(s)%2==1`であれば上記の最終状態まで偶数回の操作が必要
          - よって'Second'が勝つ
    - `s[0] != s[-1]` の時
      - `'ababab'` 的な長さ **偶数** の文字列になって終わる
      - あとは同様
